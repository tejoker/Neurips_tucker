## NeurIPS Ablation Protocol (Canonical)

This document defines the single source of truth for ablation metrics, scripts, and figure/table provenance.

### Metric Mapping (Do Not Mix)

- `Anomaly detection ablation` (used in NeurIPS main text):
  - Metrics: `PA-F1`, `Std-F1`, `AUC-PR`, `time_total_s`, `param_count`
  - Script: `python -m ablation.run_component_ablation --profile neurips`
  - Outputs: `results/ablation_component/ablation_raw_results.csv`, `results/ablation_component/ablation_aggregated.json`, `results/ablation_component/ablation_table.tex`
- `Rank sensitivity` (appendix/computational scaling):
  - Metrics: `PA-F1`, `Std-F1`, `param_count`, `time_total_s`
  - Script: `python -m ablation.run_rank_study`
  - Outputs: `results/ablation_rank_study/rank_study_raw.csv`, `results/ablation_rank_study/rank_study_summary.json`
- `Synthetic structural graph recovery` (optional diagnostic only):
  - Metrics: graph `precision/recall/f1` over edge tuples
  - Script path: `scripts/run_ablation_suite.sh` + `scripts/evaluate_ablation_metrics.py`
  - Must not be reported as anomaly detection F1.

### NeurIPS Table/Figure Provenance

- `Table 2 (component ablation)`:
  - Source: `results/ablation_component/ablation_aggregated.json`
  - Variants: `full`, `no_tucker`, `cp_decomposition`, `single_metric`, `l1_sparsity`
  - Statistics: mean, std, 95% CI, paired t-test vs full model
- `Rank scaling figures (appendix)`:
  - Source: `results/ablation_rank_study/rank_study_raw.csv`
  - Generation: `python scripts/analyze_ablation.py --results_dir results/ablation --output_dir paper_plots`
  - Expected files: `paper_plots/ablation_scaling_msl.pdf`, `paper_plots/ablation_heatmap_msl.pdf`, `paper_plots/ablation_scaling_highd.pdf`, `paper_plots/ablation_heatmap_highd.pdf` when both datasets are present.

### Standard Run (Server-Friendly)

```bash
bash scripts/run_neurips_ablation.sh
```

Optional knobs:

```bash
DEVICE=cpu SEEDS=5 MAX_ITER=80 DATA_ROOT=ServerMachineDataset bash scripts/run_neurips_ablation.sh
```

### Reproducibility Requirements

- Use fixed seeds from `ablation.run_component_ablation.DEFAULT_SEEDS`.
- Run at least 5 seeds for publication numbers.
- Keep dataset root explicit (`DATA_ROOT`) to avoid machine-specific paths.
- Report only files generated by committed scripts.
